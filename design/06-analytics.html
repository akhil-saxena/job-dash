A1: Dashboard Grid

Bento-style layout. Everything visible at once on desktop. Four big stat cards at top, funnel and sources side by side below, response time table at bottom.

  • All data visible without scrolling on desktop
  • Clear visual hierarchy: numbers first, charts second, details third
  • Date filter chips in header control the entire view
  • Mobile: 2x2 stat grid, stacked cards below
  • Most information-dense option

A2: Single Column

Reading-order layout. Stats as compact inline row (not big cards), then funnel, then sources, then response times. Scrollable. Centered 600px column on desktop.

  • Natural top-to-bottom reading flow
  • Compact stat row saves vertical space vs big cards
  • Identical layout between desktop and mobile — just narrower
  • Subtitles under each section provide context
  • Best for scanning quickly, worst for dashboards-at-a-glance

A3: Tabs

Three tabs: Summary (stats + mini funnel), Sources (detailed source breakdown with counts), Timeline (response times + speed distribution). Each tab gets full space.

  • Less visual clutter — only one section at a time
  • Sources tab shows detailed breakdown per source
  • Timeline tab adds speed distribution buckets
  • Underline tab bar matches app navigation style
  • Tabs work identically on mobile
  • Tradeoff: can't see everything at once
