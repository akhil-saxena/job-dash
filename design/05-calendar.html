C1: Month Grid + List

Traditional calendar layout. Full month grid at top shows event density at a glance via colored chips on date cells. Below, two glass-card list sections: "This Week" for upcoming events and "Past" for completed/overdue. Month/Week toggle in the header.

  • Best for spatial thinkers who want the month overview
  • Chips on dates show event density without detail
  • Today (18th) highlighted with dark circle
  • List sections below give readable detail for the near term
  • Mobile: same structure, compact grid then scrollable list

C2: Agenda View

No month grid. Scrollable list of upcoming events grouped by day. Each event is its own glass card with colored left bar, company name, event type, and time. Past events dimmed. Mini week strip at top shows which days have events via colored dots.

  • Fastest way to see what's next without visual noise
  • Each event card is self-contained and scannable
  • Week strip gives minimal context without a full grid
  • Today's event gets a stronger card + accent border
  • Best when most users just need "what's coming up"

C3: Split View

Desktop: month grid on left (narrower panel), scrollable event list on right (wider). Two glass panels side by side. Mobile: month grid on top (compact), event list below. Grid dots are clickable to scroll the list to that day.

  • Both month context and detail visible simultaneously
  • Grid acts as navigation, list shows detail
  • Legend at bottom of grid explains dot colors
  • Today's event card is visually prominent with accent
  • Mobile gracefully stacks the two panels vertically
