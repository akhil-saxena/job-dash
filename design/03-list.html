L1: Glass Table

Traditional table layout wrapped in a frosted glass container. Sortable column headers let users reorder by company, status, date, or days. Filter chips above act as quick status tabs.

  • Most data-dense -- 7 columns visible at once
  • Sortable headers for flexible ordering
  • Filter chips double as status tab navigation
  • Mobile: switches to card list with badge + status pill
  • Best for power users who want spreadsheet-like scanning

L2: Grouped List

Applications organized by status in collapsible sections. Each section header shows status name and count. Cards within sections are wide horizontal rows with badge, name, role, source, and days.

  • Natural mental model -- grouped by pipeline stage
  • Section headers with counts give quick overview
  • Priority ordering: Offer first, then Interviewing, etc.
  • Mobile: same grouped layout, compact cards
  • Best for users who think in terms of pipeline stages

L3: Compact Rows

Maximum density. No cards, no borders between items -- just alternating rows in a single glass container. Each row: badge, company, role, status dot, days. See all applications at a glance.

  • Highest information density of all three variants
  • Status dot instead of pill -- minimal visual weight
  • Alternating row tints for scanability
  • Mobile: same rows but 44px min-height for tap targets
  • Best for users who want the most items visible at once
