**1. Enhance `src/client/lib/urgency.ts`:**

Update the `UrgencyInput` interface to accept optional deadline data:

```typescript
interface UrgencyInput {
  status: string;
  updatedAt: number;
  deadlines?: Array<{
    deadlineType: string;
    dueAt: number;  // unix epoch seconds
    isCompleted: number;
  }>;
  nextInterviewAt?: number;  // unix epoch seconds, from interview_round.scheduled_at
}
```

Update `calculateUrgency` to check deadlines:

Priority order (first match wins):
1. `status === "rejected"` -> "rejected"
2. If nextInterviewAt is today -> "interview-today" (amber tint, per DEAD-02)
3. If nextInterviewAt is tomorrow -> "interview-tomorrow" (amber tint)
4. If nextInterviewAt is within 7 days -> "interview-week" (no tint, just for future use)
5. If any non-completed deadline with deadlineType "offer_expiry" is within 3 days -> "offer-expiring" (green tint, per DEAD-02)
6. If daysSinceUpdate >= 7 and status not terminal -> "stale" (red tint, per DEAD-03)
7. Otherwise "normal"

Helper: `daysUntil(epochSeconds: number): number` -- returns positive for future, negative for past.
