# Phase 6 Plan 1: DB Schema for Tags, Deadlines & Company

## Objective

Add Drizzle schema definitions, shared constants, Zod validators, and a Drizzle migration for the tag, application_tag, deadline, and company tables. Salary fields already exist on the application table. This plan lays the data foundation for Phase 6 features.

## Context

- @src/db/schema/application.ts (existing application table with salary fields)
- @src/db/schema/interview.ts (pattern for new tables)
- @src/shared/constants.ts (existing constants)
- @src/shared/validators/application.ts (existing validators)
- @src/shared/validators/interview.ts (pattern for new validators)
- @tests/setup.ts (inline migration SQL for tests)

## Tasks

### Task 1: Schema definitions, constants, and validators

type="auto"

**Add schema files:**

1. `src/db/schema/tag.ts` -- `tag` table (id, userId, name, color, createdAt, updatedAt) with unique index on (userId, name). `application_tag` join table (id, applicationId, tagId, userId, createdAt) with unique index on (applicationId, tagId).
2. `src/db/schema/deadline.ts` -- `deadline` table (id, applicationId, userId, deadlineType, label, dueDate, isCompleted, createdAt, updatedAt) with index on (applicationId) and (userId, dueDate).
3. `src/db/schema/company.ts` -- `company` table (id, userId, name, domain, website, notes, createdAt, updatedAt) with unique index on (userId, domain) where domain is nullable, and index on (userId, name).
4. Update `src/db/schema/index.ts` to export new schemas.

**Add constants:**
- `DEADLINE_TYPES`: application_close, offer_expiry, follow_up, custom
- `TAG_COLORS`: slate, red, orange, amber, yellow, lime, green, emerald, teal, cyan, sky, blue, indigo, violet, purple, fuchsia, pink, rose

**Add validators:**
- `src/shared/validators/tag.ts` -- createTag, updateTag, assignTag schemas
- `src/shared/validators/deadline.ts` -- createDeadline, updateDeadline schemas
- `src/shared/validators/company.ts` -- createCompany, updateCompany schemas

**Done when:**
- All schema files compile with no TS errors
- Constants exported from shared/constants.ts
- Validators exported and use constants for enums

### Task 2: Drizzle migration and test setup update

type="auto"

1. Run `npx drizzle-kit generate` to create the migration SQL.
2. Update `tests/setup.ts` to include the new tables in the inline migration SQL.
3. Run `npx vitest run` to confirm existing tests still pass with the new schema.

**Done when:**
- Migration SQL file exists in src/db/migrations/
- Test setup includes new table DDL
- All existing tests pass (no regressions)

## Verification

- `npx tsc --noEmit` passes
- `npx vitest run` passes
- Migration file generated

## Success Criteria

- tag, application_tag, deadline, company tables defined in Drizzle schema
- Constants for deadline types and tag colors exported
- Zod validators for all new entities
- Migration SQL generated
- Existing tests unbroken
