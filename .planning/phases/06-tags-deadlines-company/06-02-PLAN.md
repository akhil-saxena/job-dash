**1. Create `src/server/services/tag.ts`** with these functions:

- `create(db, userId, input: { name, color? })` -- insert into tag table. Check uniqueness (tag name per user) -- if conflict, throw ConflictError("Tag name already exists").
- `list(db, userId)` -- SELECT all tags for user, ordered by name ASC. Return array of tag rows.
- `update(db, userId, tagId, input: { name?, color? })` -- verify ownership (tag.userId === userId), update fields + updatedAt. Throw NotFoundError if not found.
- `remove(db, userId, tagId)` -- verify ownership, DELETE from tag (cascade deletes application_tag rows). Throw NotFoundError if not found.
- `assignTag(db, userId, applicationId, tagId)` -- verify both application and tag belong to userId. Insert into application_tag. If already assigned (unique constraint violation), silently return existing row (use INSERT OR IGNORE pattern or catch).
- `unassignTag(db, userId, applicationId, tagId)` -- DELETE from application_tag WHERE applicationId AND tagId. Verify application ownership.
- `getTagsForApplication(db, userId, applicationId)` -- join application_tag with tag, return tag rows for the given application. Verify application ownership.
- `getApplicationIdsByTag(db, userId, tagId)` -- SELECT applicationId from application_tag WHERE tagId. Used for filtering.

All functions take `db: Database` as first param per existing pattern. All verify tenant isolation via userId.

**2. Create `src/server/services/deadline.ts`** with these functions:

- `create(db, userId, applicationId, input: { deadlineType, customLabel?, dueAt, notes? })` -- verify application ownership. Insert into deadline. Convert dueAt string to Date. Return created row.
- `listForApplication(db, userId, applicationId)` -- verify application ownership. SELECT deadlines for app, ordered by dueAt ASC.
- `listUpcoming(db, userId)` -- SELECT all non-completed deadlines for user where dueAt >= now, ordered by dueAt ASC. For urgency calculation on the frontend.
- `update(db, userId, deadlineId, input)` -- verify ownership (deadline.userId === userId). Update fields + updatedAt. If dueAt provided, convert string to Date.
- `complete(db, userId, deadlineId)` -- set isCompleted = true, updatedAt = now.
- `remove(db, userId, deadlineId)` -- verify ownership, DELETE.

**3. Create `src/server/services/company.ts`** with these functions:

- `findOrCreate(db, userId, input: { name, domain?, website?, notes? })` -- if domain is provided, first try to find existing company with same userId + domain. If found, return it. Otherwise, insert new company. This implements COMP-01 domain dedup.
- `list(db, userId)` -- SELECT all companies for user, ordered by name ASC.
- `getById(db, userId, companyId)` -- verify ownership, return company row.
- `update(db, userId, companyId, input: { name?, domain?, website?, notes? })` -- verify ownership, update + updatedAt. This is how COMP-02 (research notes) works: update(db, userId, companyId, { notes: "markdown..." }).
- `remove(db, userId, companyId)` -- verify ownership, DELETE.
- `getByDomain(db, userId, domain)` -- find company by domain for this user. Returns null if not found.

All services use the same import pattern as existing services. Use `eq`, `and`, `desc`, `asc` from drizzle-orm.
