## Spacing Scale

Declared values (all multiples of 4, matches Phase 3 tokens):

| Token | Value | Usage in Phase 8 |
|-------|-------|------------------|
| xs | 4px | Event-chip internal padding (`px-1` gaps between chip dot + text); stat-card caption gap; day-cell padding (`p-1`) |
| sm | 8px | Stack gap between event chips inside a day cell; gap inside stat cards between value and label; gap between filter chips |
| md | 16px | Default card inner padding (`p-4`); gap between analytics panels on mobile |
| lg | 24px | Section padding on analytics page; gap between stat-card grid and first chart |
| xl | 32px | Gap between major analytics sections on desktop (funnel → source → table); page top padding under header |
| 2xl | 48px | Empty-state vertical breathing room (above/below illustration + copy) |
| 3xl | 64px | Full empty-state page height padding on `/calendar` and `/analytics` when no data exists |

**Exceptions:**
- Event chip height: **20px** (compact — below 24px is needed to fit 3 chips + day number inside ~100px mobile cell). Chip is pill-shaped, `rounded-[var(--radius-pill)]`, `h-5 px-2`.
- Month grid day cell min-height: **96px desktop / 64px mobile** (these are viewport-driven, not spacing tokens).
- Response-time table cell padding: **12px vertical / 16px horizontal** — 12px vertical reads as "between sm and md"; included because dense tabular data needs tighter vertical rhythm than default `p-4`.

All other values follow the 4/8/16/24/32/48/64 scale.

## Typography

Four sizes, two weights. Inherits `--font-sans` (Apple system stack) from `src/client/index.css`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 12px (`text-xs`) | 600 semibold | 1.4 | Stat card label ("TOTAL APPS"); filter chip text; column headers in response-time table; weekday headers (Mon/Tue/…) in month grid (uppercase, tracking-wider); event chip text; day-number badge in calendar cells |
| Body | 14px (`text-sm`) | 400 regular | 1.5 | This Week row role/company; table cell text; settings threshold hint copy |
| Heading | 20px (`text-xl`) | 600 semibold | 1.2 | Section headings on analytics page ("Pipeline", "Sources", "Response Time"); settings section title ("Analytics") |
| Display | 28px (`text-[28px]`) | 600 semibold | 1.1 | Stat card values (big numbers); month name ("April 2026") in calendar header |

**Tabular numerals:** Stat card values, response-time table cells, and conversion-% labels use `tabular-nums` (font-variant-numeric) so digits align vertically.

**Weekday headers:** Use the 12px label tier with `uppercase tracking-wider` and semibold weight — matches the existing `ColumnHeader` minimal variant already used on the kanban board. No new size tier is introduced.

**Dark-mode text:** Use `text-text-primary dark:text-dark-accent` for headings, `text-text-secondary dark:text-dark-accent/70` for body, `text-text-muted dark:text-dark-accent/40` for muted captions (tokens defined in `index.css`).

### Color-Blind-Safe Reinforcement

Color alone must NOT carry meaning. Enforced as follows:

- **Funnel conversion %:** shown as text ("43% of Applied"), not just bar length. Color is decorative.
- **Source effectiveness segments:** tooltip on hover shows the exact count and outcome label ("Offer: 3 · LinkedIn"). Segment icon/pattern not needed because tooltip is explicit.
- **Response-time cells:** every cell shows the numeric day count (e.g. "4.2d avg"). The day number is the primary signal; color reinforces. A color-blind user reading the number alone still gets the answer.
- **Calendar event chips:** every chip shows role/company text. The color dot is a reinforcement; the text is primary. Past-date opacity (`opacity-70`) is a secondary reinforcement beyond color.
- **Threshold preview dots in Settings:** accompany each input with the literal word zone label below: e.g. "Green <7d", "Amber <14d", "Red ≥14d".

## Layout Specifications

### Calendar view (`/calendar`)

**Page structure (top-to-bottom):**

1. **Page header** (existing AppShell header + new in-page sub-header):
   - In-page sub-header row: month name display (28px semibold), month nav (‹ Today ›) as ghost buttons on the right.
   - Height: 56px, `lg:px-6 px-4 py-3`.
2. **Weekday strip:** 7-column grid, 12px uppercase semibold labels with `tracking-wider` (`Sun Mon Tue Wed Thu Fri Sat`), padding `py-2`, no background. Uses the Label typography tier.
3. **Month grid:** `grid grid-cols-7 gap-px` with a 1px divider color of `rgba(0,0,0,0.06)` / `rgba(255,255,255,0.06)` dark.
   - Each cell: min-height 96px desktop, 64px mobile. `p-1` (4px padding — preserves density so the day number badge + up to 3 stacked 20px chips + gaps + "+N more" indicator all fit inside the 64px mobile cell; `p-2` would overflow). Rounded corners only on the outer 4 cells (top-left, top-right, bottom-left, bottom-right of the whole grid) — inner cells are flat.
   - Cell background: `glass` utility for in-month days, `bg-transparent` for out-of-month days with `opacity-40`.
   - Day number: top-right, 12px semibold (Label tier). Current day: day number wrapped in a `rounded-full h-5 w-5 bg-surface-accent text-white` badge (dark: `bg-dark-accent text-dark-dominant`).
   - Event chip stack: below day number, `flex flex-col gap-1`. Max 3 chips. Each chip is `h-5 rounded-[var(--radius-pill)] px-2 text-xs truncate` (12px Label tier — fits inside the 20px chip height with `px-2` padding).
   - "+N more" row: appears as the 4th row when 4+ events exist, same chip shell, text: `+{N} more`, color `text-text-secondary`, hover `bg-black/[0.04]`.
4. **This Week list:**
   - Section heading: "This Week" (20px semibold), right-aligned subtext: date range ("Apr 23 – Apr 29").
   - Content: glass `Card` containing rows grouped by day. Each day group is prefaced by a day label (e.g. "Today · Thu", "Tomorrow · Fri", "Sat, Apr 25") at 12px semibold uppercase tracking-wider.
   - Within a day: one row per event. Row layout: `flex items-center gap-3 py-2`: event color dot (8px) · time ("2:30 PM") 14px text-text-secondary tabular-nums · company badge · role title 14px · chevron-right icon (lucide `ChevronRight` 16px, `text-text-muted`).
   - Empty day in This Week: single muted row "No events" 14px text-text-muted, `py-2`.
   - Click row → same navigation behavior as chip click (D-06).
5. **Bottom padding:** 64px (`pb-16`) on mobile to clear the BottomTabBar.

**Grid always shows 6 rows (42 cells)** for consistent height — out-of-month cells at 40% opacity carry spillover days.

## Chart Visual Contracts

### Pipeline Funnel

- **Library:** Recharts `BarChart` with `layout="vertical"` + `CartesianGrid` hidden.
- **Bar width scaling:** Each bar's pixel width = `(count / maxCount) * 100%`. Even if top bar (`Applied`) has 42, its bar fills 100% of the plot area; all others scale proportionally.
- **Bar labels:** Two `<LabelList>` placements per bar:
  - `position="insideLeft"` — stage name + count, color white (ensure contrast on the bar fill). Example: `Applied · 42`.
  - `position="right"` — conversion caption, 12px text-text-secondary. Example: `43% of Applied`. Top bar says `100%`.
- **Empty funnel (no data):** render an inline empty state inside the panel instead of the chart — see Empty States.
- **Accessibility:** Each bar has `aria-label="{stage} stage: {count} applications, {pct}% conversion from {prevStage}"`.

### Source Effectiveness

- **Library:** Recharts `BarChart` with `layout="vertical"`, stacked bars via `stackId="outcome"` on each `<Bar>`.
- **Segments:** fixed order from left: Offer → Interviewing → Rejected → Ghosted → Withdrawn (see Color palette).
- **Source name:** rendered via `<YAxis type="category" width={120} />` with 12px text, left-aligned.
- **Segment labels inside bar:** only if segment count ≥ 1 AND segment width in pixels ≥ 20px. Otherwise no inline label; tooltip reveals the value.
- **Legend:** below chart, 12px semibold, 5 entries with color swatches.
- **Tooltip (glass Card styled):** on hover over a bar row, show: source name · 5 lines, one per outcome: `{dot} {Outcome}: {count}`.
- **Top-N handling:** Server returns top 8 sources by total application volume within the selected date range. If user has fewer than 8 distinct sources, render whatever exists.
- **Empty-source source chart:** see Empty States.

### Summary Stat Cards

- **Four cards**, fixed order left-to-right: Total Apps · Currently Active · Offers Received · Rejection Rate.
- **Value formats:**
  - Total Apps: integer, e.g. `42`
  - Currently Active: integer, e.g. `18`
  - Offers Received: integer, e.g. `3`
  - Rejection Rate: percent with 0 decimals, e.g. `45%`. If denominator is 0 (no terminal-state apps), show `—` with caption "No outcomes yet".
- **Caption under value (optional, only when meaningful):**
  - Rejection Rate always shows caption: `12 of 27 terminal`
  - Others no caption unless zero-state: value is `0` → caption "Try adding an application".
- **Zero-state for the whole strip:** if Total Apps = 0, still render the cards with `0` values but they link to a prominent empty page (see Empty States — no applications at all).

## Error States

### Calendar fetch fails

- Replace grid + This Week list with a single centered glass card:
  - Icon: `AlertCircle` 32px `text-status-rejected`.
  - Heading: "Couldn't load your calendar"
  - Body: "Something went wrong fetching interviews and deadlines. Check your connection and try again."
  - CTA: "Retry" (filled sm) — re-runs the query.
- Do not show partial data on error (calendar is all-or-nothing for a single month view).

### Analytics fetch fails (any endpoint)

- Per-panel error: if only one panel's query fails, replace ONLY that panel's content with a compact error:
  - Icon: `AlertCircle` 20px `text-status-rejected`.
  - Body: "Couldn't load {panel name}." + 12px muted subtext "Check connection and retry."
  - CTA: "Retry" (ghost sm) inline next to body.
- Stat cards all fail together (single endpoint): replace the whole strip with one error row matching the per-panel shape above, `col-span-4`.

### Settings threshold save fails

- Revert the input to its previous value (optimistic pattern from Phase 4 D-14/15).
- Toast: "Couldn't save thresholds. Reverted." (existing `ToastProvider` from Phase 1).

## Interaction Contracts

### Calendar interactions

- **Event chip (inside day cell):**
  - Default: `bg-{color}/10 border border-{color}/20 text-{color}`.
  - Hover: `bg-{color}/20 border-{color}/40 cursor-pointer`.
  - Focus-visible: `ring-2 ring-offset-1 ring-surface-accent/40 dark:ring-dark-accent/40`.
  - Click: navigates to `/app/{slug}` with `?tab=interviews` (for interview chip) or `?tab=overview` (for deadline chip), using TanStack Router `Link`. Per D-06.
  - Keyboard: chips are `<button>` elements in a flat tab order — `Tab` moves through chips in reading order (left-to-right, top-to-bottom of the grid).
- **"+N more" chip:**
  - Same focus + hover as event chip but neutral colored (`bg-black/[0.04]` light).
  - Click: opens a glass `Modal` with heading "{formatted day} · {N} events" and all events for that day listed as full `ThisWeekList` rows. Modal closes on Esc / backdrop click / event row click (navigating).
- **Day cell (non-chip area):**
  - Non-interactive — clicking empty space in a day cell does nothing. (Deliberate choice: no "create event" action in Phase 8 since calendar is read-only.)
  - Hover: subtle `bg-white/20` light / `bg-white/[0.03]` dark to indicate the cell boundary. No cursor change.
- **Month nav (‹ Today ›):**
  - Ghost buttons. Clicking arrows shifts month ±1. "Today" snaps back to current month.
  - Keyboard shortcuts: `←` prev month, `→` next month, `T` today. Hint line under the month name (only desktop, only on first visit): "Tip: ← → to navigate, T for today". Dismiss on any click.
- **This Week row:** full row is clickable, same navigation as chip click. Hover `bg-black/[0.03]`. Focus-visible ring identical to event chip.

### Analytics interactions

- **Date range filter chips:**
  - Active chip uses the accent (filled variant from the design system — `FilterChips` already handles active state).
  - Click a preset → immediate refetch of all four analytics queries (stat cards, funnel, sources, response time). Optimistic: show skeletons during refetch.
  - Click "Custom" → opens Modal with two native date inputs. Apply button triggers refetch. Closing without applying (via the "Keep previous range" button, Esc, or backdrop click) preserves the previously-applied range unchanged.
  - Default: All time (D-17). Last-used range persisted to `localStorage` key `jobdash:analytics:dateRange`.
- **Funnel bar hover:** tooltip (Recharts default styled via CSS) shows: `{stage}`, `{count} applications`, `{pct}% conversion`. Subtle bar fill shift on hover (`brightness(0.9)`).
- **Source bar segment hover:** segment-level tooltip shows outcome + count. Entire bar row gets a faint 1px right border on hover to indicate the row is being inspected.
- **Response time cell hover:** tooltip shows `{n} applications in this sample` (e.g. "Based on 14 applications").
- **Stat cards:** not clickable in Phase 8. (Potential drill-down deferred; see Deferred section.)

### Settings interactions

- **Threshold input:** number-only input (`type="number" min="0" max="365"`). On change, debounced save (500ms) via `useDebouncedMutate`. On blur, immediate save.
- **Preview dots:** non-interactive.
- **Reset defaults Button:** confirmation via native `confirm()` dialog: "Reset analytics thresholds to defaults?". Yes → PATCH with default values → all inputs update.

## Accessibility

### Keyboard navigation

- **Calendar grid:**
  - `Tab` enters the grid at the first event chip of the first day that has events.
  - Within the grid, `Tab` moves between chips in DOM reading order.
  - Arrow keys are NOT bound within the grid (per spec, grid is a list of chips, not a spatial grid — WCAG allows this when semantic is "list of events for month").
  - On the month header, arrow keys work as global shortcuts (see Interaction Contracts).
- **This Week list:** `Tab` moves row-by-row. `Enter`/`Space` activates the row (same as click).
- **Analytics page:**
  - `Tab` order: filter chips left-to-right → Custom button → (skip stat cards, they're not interactive) → funnel chart tooltips (not tabbable; Recharts default). Charts themselves are not tabbable in v1.
- **Settings thresholds:** standard form tab order, top-to-bottom, input-by-input.

### ARIA

- **Month grid:** `role="grid" aria-label="Calendar for {Month Year}"` on the grid container. Each day cell: `role="gridcell" aria-label="{weekday}, {formatted date}, {N} events"`.
- **Event chips:** `<button>` elements, no extra aria needed (visible text + click target is enough). Past-event chips get `aria-describedby` pointing to a visually-hidden span "Past event".
- **Funnel bars:** see Chart Visual Contracts — each bar has `aria-label="{stage}: {count}, {pct}% from {prev}"`.
- **Response-time cells:** the colored cell is a `<td>` with `aria-label="{transition}: {n} days average, {zone} zone"`. This gets the color meaning to screen readers.
- **Filter chips:** already accessible via the existing `FilterChips` component (buttons with pressed-state text contrast). Confirm `aria-pressed={isActive}` is set in the component (add if missing — this is a Phase 8 upgrade to existing component).

### Color-blind safety (reiterated)

Already covered above. Summary: (1) numbers accompany every colored cell, (2) text accompanies every chip, (3) legend shows both color + outcome name for source stacks, (4) funnel has explicit "% of X" text labels.

### Touch targets

- Event chip: 20px tall × variable width. Below WCAG 24px minimum. **Exception accepted** because the chip is embedded in a list context where the whole row/cell also reacts on mobile (use `touch-action` to make cells fully tappable, with overflow "+N more" serving as the expand gesture). If the chip is a sole mini-tap target, wrap each chip in a `py-1` padding container that brings the effective touch area to 28px.
- Filter chips: existing component already ≥ 32px (`py-1.5 px-3`) — compliant.
- Threshold inputs: default browser input height ~ 36px — compliant.

## Responsive Breakpoints

Uses existing Tailwind v4 defaults: `sm 640`, `md 768`, `lg 1024`, `xl 1280`. Phase 3 established mobile < 768 (`md`) for sidebar/bottom-tab-bar switch.

### Calendar responsive rules

| Viewport | Behavior |
|----------|----------|
| `< 640px` (sm) | Day cell min-height 64px. Show max 2 chips per cell (plus "+N more"). Weekday labels shortened to 1 letter (`S M T W T F S`), still at 12px label tier. This Week list: full-width. Month nav: arrows scale down to 28px. |
| `640–1023px` | Day cell min-height 80px. Max 3 chips per cell. Weekday labels 3-letter. |
| `≥ 1024px` (lg) | Day cell min-height 96px. Max 3 chips. Full weekday abbreviations. This Week list appears BELOW the grid (not side-by-side — spec is explicit). |

### Analytics responsive rules

| Viewport | Behavior |
|----------|----------|
| `< 640px` | Stat cards: 2×2 grid. Charts: full-width. Response-time table: already 2 columns, fits. Filter bar wraps chips to 2 rows if needed. |
| `640–1023px` | Stat cards: 4-col grid. Charts: full-width single column. |
| `≥ 1024px` | Stat cards: 4-col grid. Charts: single column (we do NOT go 2-col — readability prevails over density). `max-w-5xl mx-auto` container to prevent over-wide charts on ultrawide screens. |

### Settings responsive rules

| Viewport | Behavior |
|----------|----------|
| `< 640px` | Threshold rows stack (label above inputs). Preview dots move below inputs. |
| `≥ 640px` | Inline row layout as specified above. |
