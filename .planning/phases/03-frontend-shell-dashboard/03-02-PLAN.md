**Step 10 — Rewrite src/client/main.tsx (per Research Pattern 3, Pitfall 4):**
- Import `RouterProvider`, `createRouter` from `@tanstack/react-router`
- Import `QueryClient`, `QueryClientProvider` from `@tanstack/react-query`
- Import `routeTree` from `./routeTree.gen`
- Import `authClient` from `./lib/auth-client`
- Import `ToastProvider` from `./components/ui/Toast`
- Import `./index.css`
- Create QueryClient with `staleTime: 60_000, retry: 1`
- Create router with `routeTree`, default context `{ auth: { isAuthenticated: false, user: null } }`, `defaultPreload: 'intent'`
- Declare `Register` module augmentation for type safety
- `InnerApp` component: call `authClient.useSession()`. While `isPending`, show a centered loading indicator (can be simple text or skeleton). Once resolved, render `<RouterProvider router={router} context={{ auth: { isAuthenticated: !!session?.user, user: session?.user ?? null } }} />`
- Root render: `<StrictMode><QueryClientProvider client={queryClient}><ToastProvider><InnerApp /></ToastProvider></QueryClientProvider></StrictMode>`
- CRITICAL (Pitfall 4): RouterProvider must ONLY render after session is resolved. This prevents the race condition where beforeLoad fires before auth state is known.
