Create `src/server/services/application.ts` with the following functions. All functions receive `db: Database` and `userId: string` as first two args. Every query MUST include `eq(application.userId, userId)` for tenant isolation per D-16.

Import from drizzle-orm: `eq`, `and`, `isNull`, `like`, `desc`, `asc`, `sql`, `count`.
Import from schema: `application`, `timelineEvent`.
Import `nanoid` from "nanoid".
Import `generateBaseSlug` from `@/shared/slug`.
Import `NotFoundError` from `@/server/lib/errors`.
Import `Database` type from `@/server/lib/db`.

**Helper: baseConditions(userId: string)** -- Returns `[eq(application.userId, userId), isNull(application.deletedAt)]`. Used in every query except restore/list-deleted. Prevents soft-delete leak per Pitfall 3.

**Helper: createTimelineEvent(db, params: { applicationId, userId, eventType, description, metadata? })** -- Returns a prepared Drizzle insert statement (NOT executed). This allows callers to batch it with other statements via `db.batch()`. Uses `nanoid()` for id, current Date for occurredAt.

**Helper: resolveUniqueSlug(db, userId, baseSlug)** -- Per D-14/D-15. Query existing slugs matching `${baseSlug}%` for this userId. If baseSlug is available, return it. Otherwise increment suffix: `baseSlug-2`, `baseSlug-3`, etc.

**create(db, userId, input):**
1. Generate slug via `generateBaseSlug(input.companyName, input.roleTitle)`, then `resolveUniqueSlug`.
2. Convert `input.appliedAt` from ISO string to Date if present.
3. Use `db.batch()` to atomically: (a) insert into application table with all fields, returning the row, and (b) insert a "created" timeline event per D-09 with description "Application created" and metadata `{ status: input.status || "wishlist" }`.
4. Return the created application from batch result index 0.

**getById(db, userId, appId):**
1. Select application WHERE id = appId AND userId AND deletedAt IS NULL.
2. If not found, throw NotFoundError.
3. Also select timeline events WHERE applicationId = appId, ordered by occurredAt desc.
4. Return `{ ...app, timeline: events }`.

**list(db, userId, params):**
1. Build dynamic WHERE conditions from params using `baseConditions(userId)` plus optional: status, priority, source filters as exact match; search as LIKE on companyName and roleTitle.
2. If `params.archived` is true, OMIT the `isArchived = false` condition; otherwise add `eq(application.isArchived, false)` to exclude archived per D-18.
3. Count total matching rows (separate query).
4. Select with ORDER BY (params.sort, params.order -- default created_at desc) and LIMIT/OFFSET for pagination.
5. Return `{ items, total, page: params.page, limit: params.limit }`.

**update(db, userId, appId, input):**
1. Find app using baseConditions + id match. Throw NotFoundError if missing.
2. Detect if `status` field is changing -- if so, prepare a status_change timeline event per D-08 with metadata `{ from: currentStatus, to: newStatus }`.
3. Detect if `isPinned` is changing -- prepare pinned/unpinned event per D-09.
4. Detect if `isArchived` is changing -- prepare archived/unarchived event per D-09.
5. Always set `updatedAt: new Date()`.
6. If timeline events needed, use `db.batch()` for atomic update + event inserts. Otherwise single update.
7. Return updated application (use `.returning()`).

**changeStatus(db, userId, appId, newStatus):**
1. Find app using baseConditions + id. Throw NotFoundError if missing.
2. If status unchanged, return app as-is (no-op per research).
3. Use `db.batch()` to atomically: (a) update status + updatedAt, returning row, and (b) insert status_change timeline event per D-08 with description `"Status changed from ${old} to ${new}"` and metadata `{ from: old, to: new }`.
4. Return updated application.

**togglePin(db, userId, appId):**
1. Find app using baseConditions + id. Throw NotFoundError if missing.
2. Flip `isPinned` value.
3. Use `db.batch()`: update isPinned + updatedAt returning row, insert pinned/unpinned event per D-09.
4. Return updated application.

**toggleArchive(db, userId, appId):**
1. Find app using baseConditions + id. Throw NotFoundError if missing.
2. Flip `isArchived` value.
3. Use `db.batch()`: update isArchived + updatedAt returning row, insert archived/unarchived event per D-09.
4. Return updated application.

**softDelete(db, userId, appId):**
1. Find app using baseConditions + id. Throw NotFoundError if missing.
2. Use `db.batch()`: set deletedAt + updatedAt returning row, insert "deleted" timeline event per D-09.
3. Return updated application.

**restore(db, userId, appId):**
1. Find app WHERE id = appId AND userId AND deletedAt IS NOT NULL (must be currently deleted).
2. Throw NotFoundError if missing.
3. Use `db.batch()`: clear deletedAt (set to null) + update updatedAt returning row, insert "restored" timeline event per D-09.
4. Return updated application.

CRITICAL: For `db.batch()`, each element must be a prepared Drizzle query object. The `.returning()` method makes the update/insert return the affected row. Destructure batch results by position: `const [updatedRows] = await db.batch([updateQuery, insertQuery])`. The update with `.returning()` gives an array of rows at index 0; timeline insert result at index 1 can be ignored.

Each route handler follows this pattern:
1. Get `userId` from `c.get("userId")` (injected by requireAuth middleware)
2. Get `db` from `createDb(c.env.DB)`
3. Get validated input from `c.req.valid("json")` or `c.req.valid("query")`
4. Call service function
5. Return response using `success()` or `paginated()` helper
6. Wrap in try/catch: if error is AppError, return `c.json(apiError(e.code, e.message), e.statusCode)`; otherwise return 500

**Endpoints (per D-07, CONTEXT.md decisions):**

1. `POST /api/applications` -- zValidator("json", createApplicationSchema) -- calls `applicationService.create(db, userId, input)` -- returns `c.json(success(result), 201)` per D-04

2. `GET /api/applications` -- zValidator("query", listApplicationsSchema) -- calls `applicationService.list(db, userId, params)` -- returns `c.json(paginated(result.items, result.page, result.limit, result.total))`

3. `GET /api/applications/:id` -- calls `applicationService.getById(db, userId, id)` -- returns `c.json(success(result))`

4. `PATCH /api/applications/:id` -- zValidator("json", updateApplicationSchema) -- calls `applicationService.update(db, userId, id, input)` -- returns `c.json(success(result))` per D-04/D-07

5. `PATCH /api/applications/:id/status` -- zValidator("json", statusChangeSchema) -- calls `applicationService.changeStatus(db, userId, id, input.status)` -- returns `c.json(success(result))` per D-07

6. `PATCH /api/applications/:id/pin` -- calls `applicationService.togglePin(db, userId, id)` -- returns `c.json(success(result))`

7. `PATCH /api/applications/:id/archive` -- calls `applicationService.toggleArchive(db, userId, id)` -- returns `c.json(success(result))`

8. `DELETE /api/applications/:id` -- calls `applicationService.softDelete(db, userId, id)` -- returns `c.json(success(result))` (returns the soft-deleted resource per D-04)

9. `PATCH /api/applications/:id/restore` -- calls `applicationService.restore(db, userId, id)` -- returns `c.json(success(result))`

10. `GET /api/applications/:id/timeline` -- calls `applicationService.getTimeline(db, userId, id)` (add this to service if not present -- queries timeline_event WHERE applicationId AND userId, ordered by occurredAt desc) -- returns `c.json(success(events))`
