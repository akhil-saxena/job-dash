**1. Create document service** (`src/server/services/document.ts`):
Follow the interview.ts service pattern exactly:

```typescript
import { eq, and, count, asc } from "drizzle-orm";
import { nanoid } from "nanoid";
import { document, application } from "@/db/schema";
import { NotFoundError, ValidationError } from "@/server/lib/errors";
import type { Database } from "@/server/lib/db";
import { MAX_FILE_SIZE, ALLOWED_MIME_TYPES } from "@/shared/validators/document";
```

Functions to implement:

- `verifyApplicationOwnership(db, userId, applicationId)` -- same pattern as interview.ts (check application exists, belongs to user, not soft-deleted). Can import from a shared location or duplicate -- duplicate is simpler.

- `verifyDocumentOwnership(db, userId, docId)` -- check document exists and belongs to user.

- `upload(db, bucket: R2Bucket, userId, applicationId, file: File, label?: string)`:
  1. Verify application ownership
  2. Validate file size (<= MAX_FILE_SIZE = 10MB) -- throw ValidationError if exceeded
  3. Validate MIME type is in ALLOWED_MIME_TYPES -- throw ValidationError if not
  4. Generate R2 key: `docs/${userId}/${applicationId}/${nanoid()}-${file.name}`
  5. `await bucket.put(r2Key, file.stream(), { httpMetadata: { contentType: file.type } })`
  6. Insert document row into D1 with all metadata
  7. Return the inserted document row

- `list(db, userId, applicationId)`:
  1. Verify application ownership
  2. Select all documents where applicationId matches and userId matches, ordered by createdAt asc
  3. Return array of document rows

- `countDocs(db, userId, applicationId)`:
  1. Return `{ count: N }` for tab badge (same pattern as interview countRounds)

- `download(db, bucket: R2Bucket, userId, docId)`:
  1. Verify document ownership
  2. Get document row for r2Key and fileName
  3. `const object = await bucket.get(row.r2Key)`
  4. If object is null, throw NotFoundError("File not found in storage")
  5. Return `{ object, fileName: row.fileName, fileType: row.fileType }` (the route will stream it)

- `remove(db, bucket: R2Bucket, userId, docId)`:
  1. Verify document ownership
  2. Get document row for r2Key
  3. `await bucket.delete(row.r2Key)` -- delete from R2
  4. Delete document row from D1
  5. Return `{ deleted: true }`

Routes (use flat path pattern per Phase 4 Hono trie router decision):

- `POST /api/applications/:appId/documents` -- multipart file upload
  Parse multipart form: `const body = await c.req.parseBody()`. Get `file` from body (type File). Get optional `label` from body. Call `documentService.upload(db, c.env.DOCS_BUCKET, userId, appId, file, label)`. Return 201 with created doc.

- `GET /api/applications/:appId/documents` -- list documents
  Call `documentService.list(db, userId, appId)`. Return with `success()`.

- `GET /api/applications/:appId/document-count` -- lightweight count for tab badge
  Call `documentService.countDocs(db, userId, appId)`. Return with `success()`.

- `GET /api/documents/:docId/download` -- download file
  Call `documentService.download(db, c.env.DOCS_BUCKET, userId, docId)`. Stream the R2 object body back:
  ```typescript
  return new Response(result.object.body, {
    headers: {
      "Content-Type": result.fileType,
      "Content-Disposition": `attachment; filename="${result.fileName}"`,
    },
  });
  ```

- `DELETE /api/documents/:docId` -- delete document
  Call `documentService.remove(db, c.env.DOCS_BUCKET, userId, docId)`. Return with `success()`.

Add `app.onError` handler identical to interviews.ts pattern.
